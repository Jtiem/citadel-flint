## 3. The Feature Filter

Before building ANY new feature, ask:

> "Does this make one of our advantages stronger?"

- **Yes** → Great, proceed to planning
- **No, but it's necessary infrastructure** → OK, but keep it minimal
- **No, it's just cool** → Don't build it. Seriously. This is how projects lose focus.

## Example (filled in)

**Problem:** AI tools generate UI code fast, but nobody checks if that code matches the company's design system or accessibility standards.

**Advantage 1:** We use math (color science) to detect visual drift, not guessing.
- Hard to copy because: it requires deep domain expertise in perceptual color models.

**Advantage 2:** We work inside any AI coding tool, not just one editor.
- Hard to copy because: we built on an open protocol (MCP) rather than editor-specific plugins.

**Advantage 3:** Our checks are deterministic — same input, same result, every time.
- Hard to copy because: most competitors use AI to review code, which gives different answers each time.
