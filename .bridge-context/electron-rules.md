# Electron Architecture & Security Rules

Bridge relies on a strict separation of concerns to prevent malicious code generated by the AI from accessing the user's local file system.

## 1. Process Separation is Absolute
* **Main Process (`electron/` folder):** This is the ONLY place where Node.js APIs (`fs`, `path`, `http`) and `better-sqlite3` can be used.
* **Renderer Process (`src/` folder):** This is standard, sandboxed React. It has NO access to Node.js. It cannot require `fs` or `sqlite3`.

## 2. Context Isolation & IPC
* `contextIsolation` MUST be set to `true` in the `BrowserWindow` web preferences.
* `nodeIntegration` MUST be set to `false`.
* You cannot use `ipcRenderer.send()` or `ipcRenderer.invoke()` directly inside React components.

## 3. The Preload Bridge (`electron/preload.ts`)
* All communication between React and Node.js must pass through explicitly defined functions using `contextBridge.exposeInMainWorld`.
* The API must be exposed as `window.bridgeAPI`.
* **Example of a valid Bridge:**
    ```typescript
    contextBridge.exposeInMainWorld('bridgeAPI', {
        // Safe, specific function
        saveComponent: (data: ComponentPayload) => ipcRenderer.invoke('db:save-component', data),
        // Listeners must be wrapped safely
        onFigmaPayload: (callback: (data: FigmaData) => void) => {
            ipcRenderer.on('figma-data-received', (_event, data) => callback(data));
        }
    });
    ```
* You MUST declare a global TypeScript interface for `window.bridgeAPI` in a `.d.ts` file so the React frontend has perfect autocomplete and type safety.

## 4. The Ingestion Server (`electron/ingestion-server.ts`)
* The local Node.js server (`http://127.0.0.1:4545`) runs on the Main process. 
* When it receives a payload, it writes to `better-sqlite3` locally, and THEN sends an IPC message to the Renderer via `webContents.send()` to trigger a UI update on the Canvas.

## 5. Content Security Policy (CSP) Exceptions
The `index.html` must include a strict CSP, but it MUST include explicit exceptions for Sandpack to function properly. The meta tag must look roughly like this:
`<meta http-equiv="Content-Security-Policy" content="default-src 'self'; script-src 'self' 'unsafe-eval' 'unsafe-inline' https://sandpack-bundler.codesandbox.io; frame-src 'self' https://sandpack-bundler.codesandbox.io blob:; worker-src 'self' blob:;">`

## 6. AI Execution Boundary
Under no circumstances should `@anthropic-ai/sdk` be imported into the `src/` directory. All AI logic, prompt construction, and API key management lives in the `electron/` backend.
