# Demo 06 — Macro Recovery (Git Time Machine + Surgical Node Transplant)

`original-card.tsx` is the production-quality repository card at commit `a3f8b12` — complete with a metrics bar (stars, forks, open issues, updated timestamp) and a full action footer (Star, Fork, Watch, View on GitHub) with correct `aria-label` and `aria-pressed` attributes. `corrupted-card.tsx` is the same file at HEAD after an AI-assisted refactor went wrong: the developer asked the AI to "extract the language bar into its own component" but the AI issued a `git checkout` on the entire file, silently reverting two other sections that had been updated in the same commit — the `repo-card-metrics` block is gone from the AST entirely, and the footer is reduced to an empty stub, causing the governance health score to drop from 94 to 61 and triggering three `A11Y-002` violations. Bridge's RecoveryPanel identifies the regression via `bridge_debt_report`, lets the user select commit `a3f8b12` in the Git Time Machine UI, reads the historical AST via `ast:git-show`, and uses `bridge_ast_mutate` to transplant only the two missing subtrees back into `corrupted-card.tsx` — leaving the new `LanguageBar` extraction intact — in strict compliance with Commandment 11 (never `git checkout` a shared file; transplant specific nodes).
