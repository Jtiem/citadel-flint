# Demo 02 — Self-Correcting AI Orchestrator

`buggy-component.tsx` is the raw output of an AI model that hallucinated three type errors while generating a data table: a `string` literal assigned to a `number` prop, an event handler with an extra parameter not present in the prop interface contract, and a `useState` initial value that references a column key (`"created_by"`) that does not exist in the `Row` type union. Bridge's orchestrator (`electron/orchestrator.ts`) runs an in-memory TypeScript type-check on every AI output before surfacing a confirmation diff to the user — all three errors are caught with exact TS error codes (TS2322, TS2345) and the AI is sent back into a correction loop, producing a clean output that compiles with zero errors before the user ever sees a change.
