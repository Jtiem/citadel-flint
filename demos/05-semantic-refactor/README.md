# Demo 05 — Semantic Refactor (Legacy Divs to Design System)

`legacy-divs.tsx` is a profile settings panel written before the design system component library existed: every layout primitive is a bare `<div>` with a BEM-inspired className (`box`, `stack`, `flex-row`), every text node is a `<span>` with an ad-hoc className (`text-label`, `text-body`, `text-caption`), and every interactive control is built from scratch with inline styles rather than the canonical `TextField`, `SelectField`, and `Button` primitives. Running `bridge_query_registry` retrieves the typed component signatures from the registry RAG index; the AI then issues a sequence of `bridge_ast_mutate` operations to swap each raw HTML element for its design system equivalent — `<div className="stack">` becomes `<Stack>`, `<span className="text-label">` becomes `<Text variant="label">`, `<button className="btn-primary">` becomes `<Button variant="primary">` — without changing any rendered output, producing an AFTER state the governance engine can fully audit and trace.
