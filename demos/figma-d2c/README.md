# Figma Design to Code — Demo

## What this demonstrates

Paste a Figma URL, pick a UI library, get production-ready component code rendered live in Flint Glass. No install, no setup, no copy-paste.

The D2C pipeline chains three steps automatically:

1. Figma payload ingestion (HydroPaste engine parses the Figma node tree into a typed AST)
2. Token extraction and mapping (DTCG tokens derived from fills, strokes, typography, spacing)
3. Library-specific code emission (shadcn, MUI, PrimeReact, or pure Tailwind)

The result appears in LivePreview within Flint Glass immediately — and GovernanceOverlay audits it for Mithril + WCAG violations on the same render cycle.

---

## Quick Start (3 commands)

These are MCP tool calls you issue from your IDE (Claude Code, Cursor, VS Code with the Flint extension).

```
# 1. Set your target library (MUI is the project default)
flint_set_library  library="mui"

# 2. Generate from a Figma URL
#    (share a Figma link and say "Build this with MUI" — the MCP routes it automatically)
flint_design_to_code  figmaPayload=<paste JSON>  library="mui"

# 3. The generated page appears in Flint Glass LivePreview
#    GovernanceOverlay fires immediately — any Mithril or a11y violations surface inline.
```

No Figma account? Use the pre-built payloads in `payloads/` — see the "Offline Demo" section below.

## Offline Demo (no Figma account required)

The `payloads/` directory contains three pre-built Figma node trees. Pass any of them directly to `flint_design_to_code`:

| Payload | Description | Complexity |
|---------|-------------|------------|
| `payloads/hero-banner.json` | Headline, description, CTA button, background | Simple |
| `payloads/account-settings.json` | ProfileCard, Tabs, form fields, ActionButtons | Medium |
| `payloads/landing-page.json` | NavBar, Hero, Features grid (3 cards), Footer | Full page |

Example — generate the Hero Banner as shadcn:

```bash
# From the project root, using the node MCP client:
node -e "
  const fs = require('fs');
  const payload = fs.readFileSync('demos/figma-d2c/payloads/hero-banner.json', 'utf8');
  // pass payload to flint_design_to_code via your MCP client
  console.log('Payload loaded:', JSON.parse(payload).name);
"
```

Or via the shell script in this directory:

```bash
./run-demo.sh payloads/hero-banner.json shadcn
./run-demo.sh payloads/account-settings.json mui
./run-demo.sh payloads/landing-page.json tailwind
```

## How the Pipeline Works

```
Figma URL
   |
   v
get_design_context  (Figma MCP -- reads node tree)
   |
   v
flint_design_to_code
   |-- HydroPasteEngine.generateJSXWithEmitter()
   |     classifyFrame()     -- card / form / nav / section / container
   |     classifyComponent() -- Input / Select / Avatar / Badge / Tabs / ...
   |     LibraryCodeEmitter  -- emits library-specific JSX
   |
   |-- Token extraction (fills, strokes, typography, spacing)
   |-- flint_map_tokens -- DTCG → shadcn CSS vars / MUI createTheme / Tailwind config
   |
   v
ComponentResult { name, code, imports, tokenRefs }
   |
   v
Flint Glass LivePreview  (srcdoc iframe, offline)
   |
   v
GovernanceOverlay audit  (Mithril + WCAG 2.1 AA, real-time)
```

## Supported Libraries

| Library | Import convention | Theme format |
|---------|------------------|--------------|
| `shadcn` | `@/components/ui/*` | CSS variables in `globals.css` |

| `primeng` | `primereact/*` | PrimeReact theme tokens |
| `tailwind` | No component library | Tailwind utility classes only |

## Token Extraction Workflow

If you want to extract design tokens from the Figma payload for use in your project:

```
# Step 1: Extract proposals (READ-ONLY — no writes)
flint_extract_tokens  figmaPayload=<JSON>

# Review the proposals — each has a confidence score and usage count.

# Step 2: Approve the tokens you want to keep
flint_approve_tokens  tokens=[...approved subset...]  projectRoot="."

# Approved tokens are written to .flint/design-tokens.json
# Governance event is recorded for full provenance.
```

## Governance Guarantees

Every component generated by this pipeline is:

- **Mithril-safe by default** — token-derived colors only, no hardcoded hex
- **a11y-gated** — WCAG 2.1 AA violations block export (form labels, ARIA, contrast)
- **Provenance-tracked** — the Figma URL and timestamp are embedded in every generated file header
- **Auditable** — run `flint_audit` on any expected-output file to see its compliance score
