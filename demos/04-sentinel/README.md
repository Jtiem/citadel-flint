# Demo 04 — Bridge Sentinel (UX Law Enforcement)

`violating-ux.tsx` is an order management screen that an AI produced when given only a feature specification and no UX constraints — it is technically complete but cognitively overwhelming. The Bridge Sentinel persona (`bridge-mcp/src/prompts/sentinel.ts`) evaluates the component against a library of UX laws and flags four violations: a ten-button toolbar with no grouping or overflow menu (Hick's Law — decision time grows with the number of choices), sixteen form fields rendered simultaneously with no progressive disclosure (Miller's Law — working memory handles 7±2 items), completely flat typography where labels, values, headings, and helper text are visually indistinguishable, and a flat `<div>` tree with no ARIA landmark regions for keyboard navigation. The Sentinel returns a structured rejection with a remediation plan specifying which fields should move behind a "Show advanced options" disclosure, which toolbar actions should collapse into an overflow menu, and what heading hierarchy to establish.
